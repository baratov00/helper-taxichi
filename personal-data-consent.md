# Что нужно для подключения оплаты TaxiChi

## ЮKassa

Нужно получить в кабинете ЮKassa:

- shopId / идентификатор магазина;
- secret key / секретный ключ API;
- return_url — куда вернуть пользователя после оплаты;
- webhook URL — адрес серверной функции TaxiChi для уведомлений о статусе платежа;
- включённый способ оплаты СБП в договоре/кабинете ЮKassa.

Правильная схема: приложение нажимает «Оплатить», сервер TaxiChi создаёт платеж в ЮKassa, получает confirmation_url, отдаёт его приложению, затем webhook или серверная проверка статуса активирует подписку и дату окончания.

## Robokassa

Нужно получить в кабинете Robokassa:

- MerchantLogin;
- Password #1 — для формирования подписи платежа;
- Password #2 — для проверки уведомлений ResultURL;
- ResultURL — серверный адрес для уведомления об успешной оплате;
- SuccessURL — куда вернуть пользователя после оплаты;
- FailURL — куда вернуть пользователя после ошибки;
- флаг тестового/боевого режима;
- включённый способ СБП, если оплата нужна именно по СБП.

Правильная схема: сервер TaxiChi формирует ссылку оплаты с подписью, Robokassa отправляет уведомление на ResultURL, сервер проверяет подпись Password #2, после чего активирует подписку.

## Важно для Play Market и RuStore

Если цифровой доступ продаётся внутри приложения, для Google Play обычно нужно использовать Google Play Billing для сборок Google Play, если политика Google не даёт исключения. Для RuStore можно использовать RuStore Pay или внешний провайдер согласно правилам площадки и типу товара. Перед публикацией нужно проверить выбранный способ оплаты по правилам конкретного магазина.
